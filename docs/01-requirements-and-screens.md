# 1단계: 요구사항 정리 + 기능 명세 + 화면 구조 (IA)

본 문서는 “Binance/Bybit 여러 계정의 자동매매 성과를 한눈에 보여주는 웹사이트”의 **운영 가능한 아키텍처**를 전제로, 기능/화면/데이터 요구를 고정한다.

---

## 핵심 원칙 (필수)

- **API 키/시크릿은 절대 프론트로 전달하지 않는다.** 서버에서만 거래소 API 호출.
- **Read-only 중심**: 주문 실행/포지션 변경은 하지 않으며 조회/성과 집계에 집중.
- **부분 실패 내성**: 특정 계정/거래소 API 실패해도 전체 앱은 정상 동작(성공한 데이터만으로 렌더).
- **정규화된 공통 데이터 모델**: 프론트는 거래소 차이를 모르게 설계.
- **운영 수준**: 성능(캐시/재시도/타임아웃), 유지보수(모듈화), 확장성(어댑터 패턴), 보안(서버 전용, 입력 검증) 포함.

---

## 사용자/사용 시나리오

- 여러 거래소/여러 계정의 자동매매 성과를 **통합**해서 보고 싶다.
- 특정 거래소/특정 계정만 **비교/필터**해서 보고 싶다.
- 기간(7/30/90/전체)과 날짜 범위로 성과를 **분해**해 보고 싶다.
- 대시보드는 “한눈에 이해”가 최우선이며, 상세는 거래내역/분석 화면에서 본다.

---

## 기능 명세 (MVP → 운영 확장)

### A. 데이터 수집/정규화 (서버)
- **계정 자동 등록**: env에 등록된 키를 파싱해 계정 리스트 생성.
  - 예: `BINANCE_API_KEY`, `BINANCE_API_KEY_JK`, `BINANCE_API_KEY_JK_2` … 패턴 지원
  - 메타데이터: `accountId`, `exchange`, `label`, `isEnabled`
- **거래소 어댑터**: `BinanceAdapter`, `BybitAdapter`가 동일 인터페이스 구현
  - 조회 범위:
    - **계좌 자산(Equity/Balance) 스냅샷**
    - **체결/거래 내역(실현 PnL이 있는 형태)**
  - 공통 출력:
    - `NormalizedTrade[]`
    - `EquityPoint[]` (시계열)
    - `AccountSummary` (총 손익/수익률/승률/거래수/최근거래 등)
- **캐시/재시도/타임아웃**
  - 요청 단위 서버 캐시(예: 10~30초) + 강제 새로고침 옵션
  - 계정별 타임아웃(예: 8~12초) 및 실패 격리

### B. 대시보드 (프론트)
- **통합 성과(최상단 KPI)**
  - 총 손익(PnL), 총 수익률(%), 승률, 거래 횟수, 최근 거래 시간, 마지막 업데이트 시간
- **핵심 그래프(가장 중요)**
  - 누적 수익률(라인)
  - 자산 추이(라인/에어리어)
  - 일별 손익(바)
- **비교 카드**
  - 거래소별, 계정별 성과 카드(정렬 가능)
- **필터**
  - 전체 / Binance / Bybit / 특정 계정
  - 기간 프리셋(7/30/90/전체) + 커스텀 날짜 범위

### C. 거래 내역 화면
- 테이블: 체결 시간, 심볼, 방향, 수량, 진입/청산가, 실현손익, 수익률, 거래소, 계정명
- 정렬/필터/검색(심볼/계정/거래소)
- 페이징(또는 가상 스크롤: 데이터가 많아질 때 대비)

### D. 성과 분석 화면
- 누적 수익률 곡선(필터 적용)
- MDD(최대 낙폭) 및 드로다운 곡선(옵션)
- 평균 손익, 최고/최악 거래, 연속 승/패
- 일별 성과 히트맵, 월별 성과 테이블

### E. 실시간 느낌(운영 UX)
- 자동 새로고침(예: 15~30초)
- 마지막 업데이트 시간 표시
- 실패 시 fallback:
  - 일부 계정 실패: 성공한 계정만 집계 + 실패 배지/툴팁
  - 전체 실패: 오류 상태 UI + 재시도 버튼

## 화면 구조 (IA) & 라우팅 (Next.js App Router)

- `/` → 대시보드(통합 성과)
- `/trades` → 거래 내역
- `/analytics` → 성과 분석
- `/settings` → (선택) 계정/표시 설정(계정 enable/label 등. 초기엔 read-only로 시작 가능)

### 공통 레이아웃
- 좌측 사이드바(데스크탑): Dashboard / Trades / Analytics / Settings
- 상단 바: 필터(기간, 날짜 범위, 계정/거래소) + 새로고침/업데이트 시간
- 본문: 카드 + 차트 + 테이블

## 데이터 정의(프론트가 받는 “공통 DTO” 목표)

프론트는 아래 형태만 이해하면 된다.

- `accounts[]`: 계정 메타(거래소/라벨/상태)
- `summary`: 선택 필터 기준 KPI
- `equitySeries[]`: 자산/누적손익/누적수익률 시계열
- `dailyPnl[]`: 일별 손익(바 차트)
- `breakdown.exchange[]`, `breakdown.account[]`: 비교 카드용 집계
- `trades[]`: 거래 내역(페이지에서 사용)
- `errors[]`: 계정별 실패 정보(부분 실패 표현)

---

## 비기능 요구

- **보안**: 서버에서만 키 사용, 응답에 키/시크릿/서명 정보 노출 금지
- **성능**: 캐시 + 병렬 호출 + 제한된 기간 조회 + pagination
- **유지보수**: 어댑터/정규화/계산/프론트 분리
- **확장성**: 추후 OKX/Upbit 추가 시 `Adapter`만 추가하면 되는 구조
- **관측성**: 서버 로그(에러/지연), 프론트 오류 상태 메시지 표준화
